# Metodologías Ágiles de Desarrollo

## ¿Qué es Agile?
Agile es un enfoque de desarrollo de software que se basa en la entrega
continua de valor al cliente mediante ciclos cortos de trabajo. En vez de
planificar todo al principio y entregar al final, se divide el proyecto
en pequeñas entregas funcionales que se van mejorando con el feedback
del usuario.

## ¿Qué es Scrum?
Scrum es una metodología ágil que organiza el trabajo en ciclos llamados
sprints (normalmente de 2 semanas). Sus elementos principales son:

- **Product Owner**: decide qué se construye y en qué orden
- **Scrum Master**: facilita el proceso y elimina obstáculos
- **Equipo de desarrollo**: construye el producto
- **Sprint**: ciclo de trabajo de 1-4 semanas
- **Backlog**: lista priorizada de tareas pendientes
- **Daily standup**: reunión diaria de 15 minutos
- **Sprint review**: demostración de lo construido al final del sprint

## ¿Qué es Kanban?
Kanban es un sistema visual para gestionar el trabajo. Se usa un tablero
con columnas que representan las fases del trabajo (por ejemplo: Pendiente,
En progreso, En revisión, Hecho). Las tareas se mueven entre columnas
según avanzan. La regla principal es limitar el trabajo en progreso
para evitar sobrecarga.

## Diferencias entre Scrum y Kanban

| Aspecto | Scrum | Kanban |
|---|---|---|
| Ciclos | Sprints fijos | Flujo continuo |
| Roles | Definidos (PO, SM, equipo) | No hay roles fijos |
| Planificación | Al inicio del sprint | Continua |
| Cambios | Al final del sprint | En cualquier momento |
| Tablero | Se resetea cada sprint | Es continuo |

## ¿Cuándo usar cada uno?
- **Scrum** es mejor para proyectos con entregas definidas y equipos
  que necesitan estructura clara
- **Kanban** es mejor para trabajo continuo como soporte técnico o
  proyectos donde las prioridades cambian constantemente

En MØRK Studio usaré Kanban porque al ser freelance las prioridades
cambian según los clientes y necesito flexibilidad.
